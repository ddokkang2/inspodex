# Inspodex 제품 전략서

최종 업데이트: 2026-04-11

## 1. 결론

2026년 4월 11일 기준으로, Inspodex는 지금 당장 "모든 사람을 위한 레퍼런스 사이트"나 "API 제품"으로 포지셔닝하는 것이 맞지 않다.

현재 가장 적합한 1차 방향은 아래다.

> Inspodex = 막연한 비주얼 방향을 빠르게 레퍼런스 세트와 프롬프트로 바꿔야 하는 1인 크리에이터와 프리랜서 디자이너를 위한 AI-친화형 비주얼 레퍼런스 워크스페이스

즉, 다음 원칙을 따른다.

- 넓은 카테고리 구성은 유입 강점으로 유지한다
- 제품 메시지와 유료 가치는 하나의 명확한 작업 흐름에 집중한다
- 플랫폼 확장보다 반복 사용되는 개인 작업 흐름 가치를 우선한다
- API는 나중 옵션으로 남기되, 현재 비즈니스 중심축으로 삼지 않는다

## 2. 왜 지금은 넓은 포지셔닝이 위험한가

현재 제품의 강점은 분명하다.

- 빠른 탐색
- 넓은 카테고리 폭
- 내부 탐색에서 외부 검색으로의 즉시 전환
- 프롬프트/검색어 재사용의 출발점

하지만 넓은 포지셔닝에는 문제가 있다.

- 많은 사람이 "유용하다"고 느낄 수는 있다
- 그러나 돈을 낼 정도로 "없으면 안 된다"고 느끼는 사람은 적을 수 있다

핵심은 여기다.

- 넓은 유용성은 트래픽을 만든다
- 좁은 문제 해결은 결제를 만든다

## 3. 추천 타겟 사용자

### 핵심 ICP

AI를 적극적으로 쓰는 1인 비주얼 크리에이터와 프리랜서 디자이너

예시:

- 프리랜서 브랜드 디자이너
- 빠르게 시안 보드를 만들어야 하는 UI/프로덕트 디자이너
- AI 기반 무드보드와 비주얼 방향안을 만드는 크리에이터
- 랜딩, 제안서, 콘셉트 팩을 혼자 빠르게 만드는 1인 스튜디오

### 이들의 공통 행동

- 레퍼런스 수집 빈도가 높다
- 레퍼런스 탐색과 이미지 생성 툴 사이를 자주 오간다
- 깊은 이론적 분류보다 작업 속도를 더 중요하게 여긴다
- 실제 프로젝트 시간을 줄여주면 결제할 가능성이 높다
- 팀 관리보다 개인 작업 효율이 더 중요하다

### 핵심 Job To Be Done

> 막연한 비주얼 방향을 10분 안에 작업 가능한 레퍼런스 묶음, 검색 확장어, 프롬프트 세트로 바꿔주기

## 4. 지금 메인 타겟으로 잡지 말아야 할 대상

아래는 당장 1차 타겟으로 최적화하지 않는다.

- 학습 중심의 광범위한 학생층
- 본격적인 팀 협업 툴 수요
- 가볍게 영감만 보는 캐주얼 탐색 사용자
- 레퍼런스 데이터를 가져다 쓰려는 외부 개발자

이들은 나중에 보조 타겟이 될 수는 있어도, 첫 유료 제품을 정의하는 기준이 되면 안 된다.

## 5. 제품 포지셔닝

### 추천 포지셔닝 문장

Inspodex는 AI 기반 비주얼 작업자가 스타일, 팔레트, 라이팅, 포즈, 작가 레퍼런스를 빠르게 조합해 작업용 레퍼런스 세트와 검색/프롬프트 결과로 전환하도록 돕는 도구다.

### 메인 메시지로 피해야 할 것

다음을 전면 메시지로 쓰지 않는다.

- 방대한 레퍼런스 데이터베이스
- 올인원 영감 사이트
- 모든 디자이너와 크리에이터를 위한 서비스
- 레퍼런스 API 플랫폼

이 메시지들은 넓지만, 유료 가치가 뾰족하지 않다.

## 6. 유료 제품이 팔아야 하는 것

사용자는 단순 탐색 접근권에 돈을 내지 않는다.
대신 작업 시간을 줄여주고, 반복 사용하게 해주는 구조에는 돈을 낼 수 있다.

### 무료로 남길 가능성이 높은 것

- 공개 탐색
- 기본 필터
- 기본 외부 검색 연결
- 기본 카드 상세 보기

### 유료로 설계할 가능성이 높은 것

- 프로젝트 보드/덱
- 프로젝트 단위 저장과 정리
- 후보 레퍼런스 비교
- 번들 Export Pack
  - 검색어 묶음
  - 프롬프트 묶음
  - 팔레트 묶음
  - 레퍼런스 요약 묶음
- 최근 작업 복원
- 재사용 가능한 프로젝트 템플릿
- 조합형 워크플로우
  - 스타일 + 팔레트 + 라이팅 + 포즈 묶음

### 유료 가치의 핵심 문장

> 단순히 레퍼런스를 보는 것이 아니라, 비주얼 방향 작업을 저장하고 다시 쓰게 해주는 것

## 7. 가장 중요한 기능 베팅

첫 유료 루프는 아래 3개에 집중하는 것이 좋다.

### 1. 프로젝트 보드

이유:

- 재방문 이유를 만든다
- 제품을 단순 런처에서 작업 공간으로 바꾼다
- 실제 클라이언트/작업 흐름과 연결된다

### 2. 조합형 워크플로우

이유:

- 여러 디렉터리를 하나의 결과물 경험으로 묶어준다
- 카테고리 폭이 흩어져 보이지 않고 의도적으로 느껴지게 만든다
- 일반 이미지 검색 대비 차별점이 생긴다

### 3. Export / Prompt 작업대

이유:

- 탐색에서 생성/실행으로 바로 이어진다
- AI 기반 작업자의 행동과 잘 맞는다
- 유료 전환 포인트가 분명해진다

## 8. 당장 미뤄야 할 것

핵심 루프가 검증되기 전까지는 아래를 미룬다.

- API 상품화
- 고도화된 팀 협업 기능
- 무거운 커뮤니티/소셜 기능
- 플랫폼/마켓플레이스 전략
- 모든 카테고리를 동등하게 깊게 만족시키려는 시도

플랫폼 유혹은 크지만, 지금은 집중력을 흐릴 가능성이 더 높다.

## 9. API 전략

API는 아래 조건이 충족될 때만 다시 진지하게 검토한다.

- 데이터 모델이 지금보다 훨씬 구조적이고 풍부해질 것
- 내부 추천 로직 자체가 독립 가치가 생길 것
- 외부 툴이 Inspodex 데이터를 실제로 가져가고 싶어 할 것
- API 고객이 돈을 낼 수 있다는 신호가 있을 것

그 전에는 API가 성장 동력이 아니라 분산 요인일 가능성이 높다.

현재 권장 방침:

- API-first 사업 모델은 하지 않는다
- 대신 내부 데이터 구조는 나중 API 확장이 가능하도록 정리한다
- 우선 제품 가치를 만든다

## 10. 수익화 방향

### 현재 가장 유력한 모델

개인 창작자를 위한 Product-led Freemium SaaS

### 추천 상업 구조

- Free: 탐색과 가벼운 사용
- Pro: 프로젝트 보드, 저장/재사용, 비교, Export Pack, 고급 조합 기능

### 왜 지금 API보다 강한가

- 현재 제품 강점과 더 가깝다
- 기존 UI를 바탕으로 더 빠르게 검증할 수 있다
- 실제 사용자를 붙여 보기 쉽다
- API 사업 인프라보다 기술/운영 부담이 훨씬 낮다

## 11. 반드시 검증해야 할 질문

다음 단계에서는 아래 질문에 답해야 한다.

1. 타겟 사용자는 실제 프로젝트에서 주 1회 이상 돌아오는가?
2. 저장 보드가 반복 리서치 시간을 유의미하게 줄여주는가?
3. 프롬프트/검색 Export가 유료 전환 트리거가 되는가?
4. 사용자는 단순 접근권이 아니라 "재사용 가능한 작업 흐름"에 돈을 낼 의향이 있는가?
5. 어떤 조합 묶음이 가장 가치가 큰가?
   스타일 + 팔레트
   스타일 + 라이팅
   스타일 + 포즈
   전체 비주얼 방향 세트

## 12. 향후 4-6주 핵심 지표

넓은 트래픽 지표보다 작은 핵심 지표를 본다.

### Activation

- 3개 이상 레퍼런스를 연 사용자 비율
- 외부 검색 확장을 실행한 사용자 비율
- 보드에 1개 이상 저장한 사용자 비율

### Retention

- 7일 재방문율
- 주간 재사용 사용자 수
- 프로젝트당 저장 항목 수

### Monetization Signal

- 저장/재사용 기능 요청 수
- 보드와 Export 기능에 대한 지불 의향 수
- 실제 프로젝트 사용에서 유료 관심으로 이어지는 비율

## 13. 추천 제품 로드맵

### Phase 1: 메시지 집중

- 메인페이지를 하나의 핵심 작업 흐름 중심으로 재정렬
- 일반 레퍼런스 사이트라는 설명을 줄임
- 아이디어에서 작업 가능한 레퍼런스 팩까지의 속도를 강조

### Phase 2: 첫 유료 루프 구축

- 보드/덱을 1급 객체로 도입
- 저장, 재사용, 비교 추가
- Export Pack 생성 추가

### Phase 3: 가격과 유지율 검증

- 타겟 사용자 10-15명 테스트
- 실제 프로젝트 반복 사용 관찰
- 생활 도구처럼 들어가는지 확인

### Phase 4: 검증 후 확장

- 메타데이터 심화
- 교차 카테고리 추천 개선
- 실제 공유 수요가 있을 때만 팀 기능 검토
- 제품과 데이터 레버리지가 확인되면 그때 API 재검토

## 14. 최종 제안

가장 뾰족한 단기 방향은 아래다.

> 모두를 위한 서비스가 되려고 하지 말고,
> AI 기반으로 비주얼 작업을 하는 1인 창작자가 레퍼런스에서 프롬프트까지 가장 빠르게 넘어가게 만드는 도구가 되자.

한 줄 요약:

- 데이터는 넓게, 메시지는 좁게
- 무료는 탐색, 유료는 작업 재사용
- 지금은 제품 우선, API는 나중
